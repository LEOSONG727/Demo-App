# Samsung Contents Wallet — Concept Demo

> Samsung VD 사업부 CX팀 내부 기획 컨셉 데모.
> TV를 켜면 AI Agent가 이미 편성표를 짜놓은 세상을 보여줍니다.

---

## 서비스 개요

**Samsung Contents Wallet**은 삼성 TV가 크로스 플랫폼 AI Agent로서 사용자의 콘텐츠 소비를 자동으로 설계해주는 서비스입니다.

### 한 줄 메시지

> 더 이상 콘텐츠를 찾아다니지 않고, 내 시간표에 맞춰 TV가 준비한 순서대로 자연스럽게 봅니다.

### 소비자 가치
- TV를 켜면 “오늘 뭐 볼지”가 이미 준비되어 탐색 피로 감소
- 디스커버리 경험이 “검색/선택”에서 “시간표 기반 시청 흐름”으로 전환
- KBO 연장, 늦은 귀가, 짧은 여유 시간처럼 상황이 바뀌면 편성도 자동 조정
- 긴 콘텐츠를 억지로 끊지 않고, 짧은 빈 슬롯에는 YouTube나 Samsung TV Plus 같은 가벼운 콘텐츠 배치
- 사용자는 편성을 고정, 미루기, 제외, 짧은 콘텐츠로 대체할 수 있어 자동화에 대한 통제감 확보

### VD 사업 관점
- TV Home을 단순 런처가 아닌 **생활 편성 Agent**로 확장
- Samsung TV Plus를 빈 슬롯 대체 콘텐츠로 자연스럽게 노출
- 파트너 OTT에는 찜 콘텐츠 소비와 신규 공개작 전환 기회 제공
- Galaxy, SmartThings, Knox를 결합해 OTT 단독 추천과 다른 삼성만의 UX 차별화 확보

### 핵심 문제 인식
- 사람들은 OTT를 평균 4.2개 구독하지만, 무엇을 볼지 고르는 데 평균 23분을 소비
- 찜해놓은 콘텐츠의 47%는 끝내 시청하지 않음
- 각 OTT 플랫폼은 자사 콘텐츠만 추천 → 크로스 플랫폼 큐레이션 불가

### 삼성 TV만이 할 수 있는 이유
Netflix, YouTube, Disney+, Tving, KBO 등 모든 플랫폼 **위에** 있는 삼성 TV는:
- 어떤 단일 OTT 플랫폼도 갖지 못한 **크로스 플랫폼 데이터 통합** 가능
- Galaxy 스마트폰 생활 패턴으로 **정확한 시청 가능 타임슬롯 감지** 가능
- 사용자가 직접 찾아다니는 것이 아닌, **Agent가 알아서 편성** 가능

### 제품 신뢰 장치
- **Opt-in 기반 연동**: Galaxy 생활 리듬, OTT 찜 목록, KBO 일정은 사용자가 허용한 범위에서만 사용
- **On-device 요약**: 원본 위치나 활동 로그가 아니라, 귀가/여유 시간/취침 예상 같은 요약 신호를 편성에 활용
- **Knox 보안 메시지**: 민감 데이터는 보안·동의 맥락 안에서 설명
- **Editable schedule**: 사용자는 특정 슬롯 고정, 미루기, 짧은 콘텐츠로 대체, 편성 제외를 선택 가능
- **Explainable Agent**: 각 편성 셀마다 왜 이 시간에 배치됐는지 이유를 제공

## 서비스 컨셉 상세

### 인풋 데이터

| 데이터 종류 | 출처 | 역할 |
|---|---|---|
| 찜 목록 / Watch Later | Netflix, YouTube, Disney+, Tving | 시청 의향 콘텐츠 수집 |
| 구독 채널 신규 영상 | YouTube | 자동 큐 추가 |
| 공개 예정 콘텐츠 | Netflix, Disney+ | 사전 예약 편성 |
| 스포츠 경기 일정 | KBO 리그 등 | 고정 스케줄 확정 |
| 생활 리듬 데이터 | Galaxy 스마트폰 | 타임슬롯 파악 |

### 타임슬롯 감지 로직 (Galaxy 데이터 활용)

삼성은 Galaxy 모바일 활동 기록을 통해 사용자의 실제 여유 시간을 정확하게 파악합니다:

- **첫 화면 확인 시간** → 기상 시간
- **GPS 이동 속도 패턴** → 출퇴근 시간대
- **홈 WiFi 재연결 시점** → 귀가 시간
- **마지막 스크롤 패턴** → 취침 시간

> 24시간이 아니라, 사람마다 다른 실제 여유 시간에만 콘텐츠를 배치합니다.

#### 라이프스타일별 차이 예시
- **직장인** → 20:00–22:30 저녁 고정 슬롯 (2.5시간)
- **자영업·재택** → 낮 + 저녁 분산 슬롯 (총 4–5시간)
- **야간형** → 21:00–01:00 심야 집중 슬롯 (4시간)

### AI Agent 편성 로직

총 5가지 규칙을 순서대로 적용하여 주간 편성표를 생성합니다:

1. **고정 일정 우선 확정** — KBO 경기 등 시간이 고정된 콘텐츠를 먼저 배치하고 잠금
2. **완결 시리즈 → 주말 빈지워칭 블록** — 전 회차 공개된 시리즈는 주말 연속 시청 블록에 배정
3. **연재물 → 정기 슬롯** — 매주 정해진 요일에 공개되는 콘텐츠는 해당 요일 정기 슬롯 배정
4. **짧은 콘텐츠 → 틈새 슬롯** — YouTube 숏폼(18–42분)은 평일 저녁 짧은 여유 시간에 배치
5. **취향 가중치 반영** — 시청 완료율·이력·선호 장르를 기반으로 우선순위 조정

### 데이터 품질 → 결과 품질 (점진적 고도화)

```
Level 1 (기본)
  인풋: 플랫폼 내 찜 목록만
  결과: 단순 재생 목록 수준 (기존 OTT와 동일)

Level 2 (현재 · Samsung Contents Wallet)
  인풋: 모든 플랫폼 찜 목록 + Galaxy 생활 패턴 + KBO 일정
  결과: 나만의 맞춤 주간 편성표 자동 생성

Level 3 (미래 비전)
  인풋: 시청 이력 + 취향 학습 + 가족 구성원 분리
  결과: 찜하지 않아도 AI가 자동 발굴·편성
```

## 데모 구조

1920×1080 Samsung TV UI 스타일의 인터랙티브 HTML 데모. 총 11개 화면.

### 화면 흐름

| # | 화면 | 내용 |
|---|---|---|
| 1 | **Intro** | Samsung Contents Wallet 브랜딩 |
| 2 | **Problem** | 탐색 피로 공감 시나리오 + 통계 |
| 3 | **Samsung Advantage** | OTT 추천 한계 vs Samsung TV 차별점 비교 |
| 4 | **Platform Connection** | 플랫폼 연결 애니메이션 (Netflix·YouTube·Disney+·Tving·KBO) |
| 5 | **Contents Wallet** | 수집된 51개 콘텐츠 월렛 UI (방향키 탐색 가능) |
| 6 | **Time Slot Sensing** | Galaxy 감지 신호 4가지 + 라이프스타일별 타임바 비교 + 데이터 품질 레벨 |
| 7 | **Agent Live Scheduling** | AI가 실시간으로 편성표를 채워가는 애니메이션 (로그 + 미니 그리드) |
| 8 | **Weekly Schedule + Live Replan** | 완성된 주간 편성표 + KBO 연장/귀가 지연 신호 기반 실시간 재편성 + 셀별 Agent 설명 |
| 9 | **Today's View** | 오늘 KBO 생중계 + 이후 편성 큐 |
| 10 | **Auto Wake** | TV가 자동으로 켜지며 콘텐츠 알림 |
| 11 | **Executive View** | 소비자 가치, VD 사업 임팩트, TV Plus/파트너 확장, 출시 의사결정 포인트 |

### 핵심 데모 장면

`S8`에 진입하면 Agent가 완성된 편성표를 보여준 뒤, 몇 초 후 다음 흐름이 자동으로 재생됩니다:

1. **Signal** — KBO 경기 연장 + 사용자의 귀가 지연 감지
2. **Decision** — 22:00에 예정된 Netflix 회차를 22:40로 이동
3. **Action** — 22:00 빈 슬롯에 Samsung TV Plus KBO 하이라이트 배치
4. **Result** — 소비자는 다시 찾지 않고 이어 보고, 삼성은 빈 시간을 TV Plus 추천 기회로 전환

### 조작 방법

| 키 | 동작 |
|---|---|
| `Enter` / `Space` | 다음 화면 |
| `←` `→` | 이전/다음 화면 (화면 5·8에서는 그리드 탐색) |
| `↑` `↓` | 그리드 탐색 (화면 5·8) |
| `1`–`9`, `0` | 해당 번호 화면으로 직접 이동 |
| 클릭 | 다음 화면 |

## 실행 방법

별도 빌드 불필요. `index.html`을 브라우저에서 바로 열면 됩니다.

```bash
# 로컬 서버로 실행 (권장)
python3 -m http.server 8080
# → http://localhost:8080 접속

# 또는 그냥 파일 열기
open index.html
```

**최적 환경**: 1920×1080 디스플레이 / Chrome 또는 Edge 최신 버전.
다른 해상도에서는 자동 스케일링 적용됩니다.
